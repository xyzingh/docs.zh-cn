# <a name="how-to-define-value-equality-for-a-type-c-programming-guide"></a><span data-ttu-id="e5fb4-104">如何为类型定义值相等性（C# 编程指南）</span><span class="sxs-lookup"><span data-stu-id="e5fb4-104">How to define value equality for a type (C# Programming Guide)</span></span>

<span data-ttu-id="e5fb4-105">定义类或结构时，需确定为类型创建值相等性（或等效性）的自定义定义是否有意义。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-105">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="e5fb4-106">通常，类型的对象预期要添加到某类集合时，或者这些对象主要用于存储一组字段或属性时，需实现值相等性。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-106">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="e5fb4-107">可以基于类型中所有字段和属性的比较结果来定义值相等性，也可以基于子集进行定义。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-107">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span>

<span data-ttu-id="e5fb4-108">在任何一种情况下，类和结构中的实现均应遵循 5 个等效性保证条件（对于以下规则，假设 `x`、`y` 和 `z` 都不为 null）：</span><span class="sxs-lookup"><span data-stu-id="e5fb4-108">In either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence (For the following rules, assume that `x`, `y` and `z` are not null):</span></span>  
  
1. <span data-ttu-id="e5fb4-109">`x.Equals(x)` 返回 `true`。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-109">`x.Equals(x)` returns `true`.</span></span> <span data-ttu-id="e5fb4-110">这称为自反属性。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-110">This is called the reflexive property.</span></span>  
  
2. <span data-ttu-id="e5fb4-111">`x.Equals(y)` 返回与 `y.Equals(x)` 相同的值。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-111">`x.Equals(y)` returns the same value as `y.Equals(x)`.</span></span> <span data-ttu-id="e5fb4-112">这称为对称属性。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-112">This is called the symmetric property.</span></span>  
  
3. <span data-ttu-id="e5fb4-113">如果 `(x.Equals(y) && y.Equals(z))` 返回 `true`，则 `x.Equals(z)` 返回 `true`。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-113">if `(x.Equals(y) && y.Equals(z))` returns `true`, then `x.Equals(z)` returns `true`.</span></span> <span data-ttu-id="e5fb4-114">这称为可传递属性。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-114">This is called the transitive property.</span></span>  
  
4. <span data-ttu-id="e5fb4-115">只要未修改 x 和 y 引用的对象，`x.Equals(y)` 的连续调用将返回相同的值。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-115">Successive invocations of `x.Equals(y)` return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5. <span data-ttu-id="e5fb4-116">任何非 null 值均不等于 null。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-116">Any non-null value is not equal to null.</span></span> <span data-ttu-id="e5fb4-117">但是，CLR 会在所有方法调用上检查 null，如果 `this` 引用为 null，则会引发 `NullReferenceException`。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-117">However, the CLR checks for null on all method calls and throws a `NullReferenceException` if the `this` reference would be null.</span></span> <span data-ttu-id="e5fb4-118">因此，当 `x` 为 null 时，`x.Equals(y)` 将引发异常。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-118">Therefore, `x.Equals(y)` throws an exception when `x` is null.</span></span> <span data-ttu-id="e5fb4-119">这会违反规则 1 或 2，具体取决于 `Equals` 的参数。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-119">That breaks rules 1 or 2, depending on the argument to `Equals`.</span></span>

<span data-ttu-id="e5fb4-120">定义的任何结构都已具有其从 <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> 方法的 <xref:System.ValueType?displayProperty=nameWithType> 替代中继承的值相等性的默认实现。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-120">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="e5fb4-121">此实现使用反射来检查类型中的所有字段和属性。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-121">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="e5fb4-122">尽管此实现可生成正确的结果，但与专门为类型编写的自定义实现相比，它的速度相对较慢。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-122">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
<span data-ttu-id="e5fb4-123">类和结构的值相等性的实现详细信息有所不同。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-123">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="e5fb4-124">但是，类和结构都需要相同的基础步骤来实现相等性：</span><span class="sxs-lookup"><span data-stu-id="e5fb4-124">However, both classes and structs require the same basic steps for implementing equality:</span></span>  
  
1. <span data-ttu-id="e5fb4-125">替代[虚拟](../../language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> 方法。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-125">Override the [virtual](../../language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="e5fb4-126">大多数情况下，`bool Equals( object obj )` 实现应只调入作为 <xref:System.IEquatable%601?displayProperty=nameWithType> 接口的实现的类型特定 `Equals` 方法。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-126">In most cases, your implementation of `bool Equals( object obj )` should just call into the type-specific `Equals` method that is the implementation of the <xref:System.IEquatable%601?displayProperty=nameWithType> interface.</span></span> <span data-ttu-id="e5fb4-127">（请参阅步骤 2。）</span><span class="sxs-lookup"><span data-stu-id="e5fb4-127">(See step 2.)</span></span>  
  
2. <span data-ttu-id="e5fb4-128">通过提供类型特定的 `Equals` 方法实现 <xref:System.IEquatable%601?displayProperty=nameWithType> 接口。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-128">Implement the <xref:System.IEquatable%601?displayProperty=nameWithType> interface by providing a type-specific `Equals` method.</span></span> <span data-ttu-id="e5fb4-129">实际的等效性比较将在此接口中执行。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-129">This is where the actual equivalence comparison is performed.</span></span> <span data-ttu-id="e5fb4-130">例如，可能决定通过仅比较类型中的一两个字段来定义相等性。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-130">For example, you might decide to define equality by comparing only one or two fields in your type.</span></span> <span data-ttu-id="e5fb4-131">不会从 `Equals` 引发异常。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-131">Do not throw exceptions from `Equals`.</span></span> <span data-ttu-id="e5fb4-132">仅对于类：此方法应仅检查类中声明的字段。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-132">For classes only: This method should examine only fields that are declared in the class.</span></span> <span data-ttu-id="e5fb4-133">它应调用 `base.Equals` 来检查基类中的字段。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-133">It should call `base.Equals` to examine fields that are in the base class.</span></span> <span data-ttu-id="e5fb4-134">（如果类型直接从 <xref:System.Object> 中继承，则不要这样做，因为 <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> 的 <xref:System.Object> 实现会执行引用相等性检查。）</span><span class="sxs-lookup"><span data-stu-id="e5fb4-134">(Do not do this if the type inherits directly from <xref:System.Object>, because the <xref:System.Object> implementation of <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> performs a reference equality check.)</span></span>  
  
3. <span data-ttu-id="e5fb4-135">可选但建议这样做：重载 [==](../../language-reference/operators/equality-operators.md#equality-operator-) 和 [!=](../../language-reference/operators/equality-operators.md#inequality-operator-) 运算符。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-135">Optional but recommended: Overload the [==](../../language-reference/operators/equality-operators.md#equality-operator-) and [!=](../../language-reference/operators/equality-operators.md#inequality-operator-) operators.</span></span>  
  
4. <span data-ttu-id="e5fb4-136">替代 <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType>，以便具有值相等性的两个对象生成相同的哈希代码。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-136">Override <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> so that two objects that have value equality produce the same hash code.</span></span>  
  
5. <span data-ttu-id="e5fb4-137">可选：若要支持“大于”或“小于”定义，请为类型实现 <xref:System.IComparable%601> 接口，并同时重载 [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) 和 [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-) 运算符。</span><span class="sxs-lookup"><span data-stu-id="e5fb4-137">Optional: To support definitions for "greater than" or "less than," implement the <xref:System.IComparable%601> interface for your type, and also overload the [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) and [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-) operators.</span></span>
